# CopilotCoursePracticeTask

## Overview

This project is a small Event Planner application built primarily using AI-assisted development with GitHub Copilot Agent.

The goal of the assignment was to create a working application where at least 90% of the code is generated by AI tools while the developer mainly acts as a reviewer and validator of the generated code.

The application includes:

- multiple pages
- reusable components
- routing with Next.js App Router
- multiple layouts
- mock data for events and users
- basic CRUD functionality in the admin section
- simple test setup

The development process was driven by a sequence of prompts given to GitHub Copilot Agent. Each step below documents the prompt, the generated result, and the review decisions.

## Project Structure

The project follows a modular structure based on Next.js App Router and reusable UI components.

Key folders include:

- `app/` – application routes and pages
- `components/` – reusable UI components
- `layouts/` – page layout components
- `data/` – mock data used by the application
- `types/` – shared TypeScript types and models
- `utils/` – helper functions such as formatting and filtering
- `__tests__/` – basic automated tests for selected components and pages

### Step 4 – Code Refactoring and Improvements

**Prompt**

```
Review the entire project and refactor the codebase.

Focus on:

- improving TypeScript typings
- removing duplicated code
- improving component reuse
- simplifying complex functions
- ensuring consistent naming

Do not introduce new libraries.
Keep changes minimal.
```

**Result**

Copilot reviewed the existing codebase and applied several refactoring improvements to enhance code quality and maintainability.

The following improvements were implemented:

- Extracted reusable utility functions such as `formatDate`, `formatDateLong`, and `formatTime` into a shared `utils/format.ts` module.
- Moved event filtering logic into a reusable utility function `filterEvents` located in `utils/filterEvents.ts`.
- Improved TypeScript type safety by exporting shared types like `EventFormData`.
- Simplified several components by removing duplicated date formatting logic.
- Optimized event lookup logic using a `Map` structure in the registrations page.
- Improved consistency of imports and component typings.

**What I accepted**

All refactoring changes proposed by Copilot, since they improved readability, reduced code duplication, and made the project structure cleaner.

**What I changed**

No significant manual modifications were required.

**Why**

The refactoring improved maintainability and aligned the project with better frontend architecture practices without adding unnecessary complexity.


## Tools Used

The project was developed primarily using **GitHub Copilot Agent** as an AI-assisted development tool.

The workflow followed an AI-driven approach:

1. Define a development task using a structured prompt
2. Let Copilot generate implementation through pull requests
3. Review the generated code
4. Accept or slightly adjust the changes

The majority of the application code was generated by Copilot, while the developer focused on reviewing, validating, and organizing the generated output.

## Insights and Learnings

Using GitHub Copilot Agent significantly accelerated the development process.

Key observations during this project:

- Providing a **clear and detailed initial prompt** resulted in a well-structured project scaffold.
- Breaking the development process into **multiple focused prompts** helped guide Copilot to generate better results.
- Copilot was particularly effective at generating:
  - UI components
  - page structures
  - form logic
  - TypeScript types
  - basic test setups
- Manual review was still important to ensure:
  - consistent architecture
  - readable code
  - correct component reuse

Overall, the AI-assisted workflow made it possible to build a complete multi-page application quickly while maintaining good code structure and readability.
